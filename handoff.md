# Handoff — Silpo Nutri-Score demo

Контекст для нового чату. Повна архітектура репозиторію — у [`CLAUDE.md`](CLAUDE.md), тут лише стан і що робилось.

## Що це

Статичний HTML/CSS/JS прототип категорії "Здорове харчування" для Сільпо з фічею Nutri-Score (лейбл харчової якості A–E, зараз в даних тільки A/D) + бейджі поживних атрибутів товару (веган, безлактозне, безглютенове, багато білка/клітковини, без доданого цукру тощо — з `products.js`/`TAG_META`).

Є **три паралельні демо-версії** для порівняння дизайну на живому сайті:
- `index.html` → `product.html` — класичний лістинг (базовий, без редизайну карток).
- `index-2.html` → `product-2.html` — "plated" редизайн: сірий фон, картки-плашки, теги повним текстом; Nutri-Score-бейдж лежить оверлеєм на фото товару; на PDP теги харчування — пілюлі замість іконки+підпису.
- `index-3.html` → `product-3.html` — та сама "plated"-картка, але Nutri-Score знятий з фото і разом з тегами/рейтингом зібраний під фото, під легкою сірою лінією: Nutri-Score (з підписом "Nutri-Score") окремим рядком, теги+рейтинг — рядком нижче.

Усі три версії використовують той самий Nutri-Score-бейдж (`nutri-score-small-*.svg` у лістингу, `nutri-score-big-*.svg` на PDP) — це вже сама по собі повна шкала A-B-C-D-E з підсвіченою літерою товару, не проста "пігулка". Квадратний варіант Nutri-Score (`nutri-score-square-*.svg`) прибрано з репозиторію повністю. Маленький бейдж (`.nutri-score-badge`) — 28px заввишки (був 30px), уніфіковано для всіх версій, бо клас спільний.

Перемикач демо-версій — клік по банеру (лістинг) чи фото товару (PDP) — циклічно перемикає `1 → 2 → 3 → 1 → …`, зберігаючи `?id=` на PDP. Логіка в `common.js` (`LISTING_CYCLE`/`PRODUCT_CYCLE`).
