# Handoff — Silpo Nutri-Score demo

Контекст для нового чату. Повна архітектура репозиторію — у [`CLAUDE.md`](CLAUDE.md), тут лише стан і що робилось.

## Що це

Статичний HTML/CSS/JS прототип категорії "Здорове харчування" для Сільпо з фічею Nutri-Score (лейбл харчової якості A–E, зараз в даних тільки A/D) + бейджі поживних атрибутів товару (веган, безлактозне, безглютенове, багато білка/клітковини, без доданого цукру тощо — з `products.js`/`TAG_META`).

Є **три паралельні демо-версії** для порівняння дизайну на живому сайті:
- `index.html` → `product.html` — класичний лістинг (базовий, без редизайну карток).
- `index-2.html` → `product-2.html` — "plated" редизайн: сірий фон, картки-плашки, теги повним текстом; Nutri-Score-бейдж лежить оверлеєм на фото товару; на PDP теги харчування — пілюлі замість іконки+підпису.
- `index-3.html` → `product-3.html` — та сама "plated"-картка, але Nutri-Score знятий з фото і разом з тегами/рейтингом зібраний під фото, під легкою сірою лінією: Nutri-Score (з підписом "Nutri-Score") окремим рядком, теги+рейтинг — рядком нижче.

Усі три версії використовують той самий Nutri-Score-бейдж (`nutri-score-small-*.svg` у лістингу, `nutri-score-big-*.svg` на PDP) — це вже сама по собі повна шкала A-B-C-D-E з підсвіченою літерою товару, не проста "пігулка". Квадратний варіант Nutri-Score (`nutri-score-square-*.svg`) прибрано з репозиторію повністю. Базовий маленький бейдж (`.nutri-score-badge`) — 28px заввишки, спільний для всіх версій; варіант 1 (`.listing-classic`) override'ить до 40px.

Варіант 1 (`index.html`/`script.js`/`pdp.js`) отримав окремий раунд правок, не пов'язаний з "plated"-редизайном варіантів 2/3:
- Теги харчування (Веган/Органік/Без лактози/Без глютену/Без доданого цукру) — кольорові іконки з `docs/assets/tags/*.svg` замість сірих кружечків з кодом (код-фолбек лишився для "Багато білка"/"Багато клітковини", бо іконок для них нема). Іконка кропиться в коло через `object-fit:cover`, для `organic.svg` є `object-position`-корекція (glyph зміщений від центру полотна source SVG), і сама іконка рендериться на 2px більшою за маску кола (`calc(100% + 2px)` + `max-width:none`, бо глобальний `img{max-width:100%}` інакше обрізає лише по ширині).
- Nutri-Score-бейдж (40px, big-асет) перенесений на фото товару, у лівий нижній кут, як промо-бейджі.
- Теги (32px) лишились у контентній частині картки, поруч із рейтингом (не на фото).
- Важливий патерн, що виринув двічі: тултіп (`::after` на `.has-tooltip`) не можна вішати на той самий елемент, що має `overflow:hidden` (потрібен для кола/скруглених кутів фото) — він сам себе обріже. Рішення — обрізку виносити в окремий вкладений wrapper, а `.has-tooltip` лишати `overflow:visible`. Так само довелось зняти `overflow:hidden` з `.product-image` (замінивши на `border-radius` прямо на `<img class="photo">`) для варіанту 1, коли Nutri-Score переїхав на фото.

Перемикач демо-версій — клік по банеру (лістинг) чи фото товару (PDP) — циклічно перемикає `1 → 2 → 3 → 1 → …`, зберігаючи `?id=` на PDP. Логіка в `common.js` (`LISTING_CYCLE`/`PRODUCT_CYCLE`).
